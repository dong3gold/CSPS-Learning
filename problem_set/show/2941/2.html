6417. 不可做的题(impossible)
(File IO): input: impossible.in output: impossible.out

Time Limits: 1000 ms Memory Limits: 528400 KB

Description

【题目背景】
小C为CZOI出了一道压轴题，出完之后却发现自己也不会做，于是完成std的任务就交给了你。
【题目描述】
现在有一张 n个点，m 条边的带权连通图，边权非负。对每一条边，计算有多少条经过这条边的从1到n的最短路。

Input

第一行， n 和 m.
接下来 m 行，每行三个正整数u ,v , w，表示一条连接节点 u和 v，权值为 w 的边。

Output

m行，按输入顺序给出经过每条边的最短路数量。由于答案可能很大，请将答案对10^7+7 取模后输出。如果存在无穷多条经过这条边的最短路，输出inf

Sample Input 

样例输入1
4 5
1 2 1
1 3 2
1 4 3
2 4 2
3 4 1


样例输入2
4 4
1 2 1
2 3 0
3 4 1
1 4 2

Sample Output 

样例输出1
1
1
1
1
1
【样例解释1】
最短路径：1->2->4,1->3->4,1->4


样例输出2
inf
inf
inf
1
【样例解释2】
可以不断在2，3之间的边来回走

Data Constraint

对所有数据，保证 ，无重边自环，顶点1,n不与零权边直接相连。


特殊性质A：无零权边
特殊性质B：m=n-1
特殊性质C：保证最短路唯一

提示：本题完全可以在O(nlogn)的时间内通过，数据范围只是为了帮助你们AK。
