6420. 【NOIP2019模拟2019.11.10】迷雾华光
(File IO): input: mode.in output: mode.out

Time Limits: 10000 ms Memory Limits: 524288 KB

Description

众所周知，在一场考试中，三道题是相对独立的，这是因为题目考察的知识点往往不同，不同的知识点往往又具有一定独立性。因此解决一道题目与掌握这道题目涉及的算法知识密切相关。
小L想知道他最应该学习什么，小L的请求用我们所学的OI模型来描述大概是这样的：
1. 将比赛看作一棵n个点的树形结构，每个点代表一场比赛且具有三个数
(ai,1,ai,2,ai,3)，这对应这场比赛中三道题涉及的知识点；
2. 小L的竞赛历程可以视作树上的一条路径 (u,v)，你需要求出小L最应该学习什么知识，即这条路径上出现次数最多的数是什么（一个点上如果有相同的数算作出现多次）；
3. 由于世界线的变动，小 L 的竞赛历程可能有微小的变化，因此有多次询问，这里用q表示询问次数。
请你完成小L的请求。

Input

从文件 mode.in 中读入数据。
输入第一行包含一个整数 Type 表示强制在线参数，它的作用在下文中会给出解释。
接下来一行包含一个正整数 n ，意义如题目描述中所示。
接下来 n 行，每行三个整数分别为ai,1,ai,2,ai,3。
接下来n-1行，每行两个整数x,y，描述比赛之间的关系，也就是树的一条边，对于所有数据，1号点为树的根。
再接下来的一行是一个整数q，意义如题目描述中所示。
接下来q行，每行两个整数u,v，表示本次询问路径的两个端点。记 xorsum 为前前面询问答案的异或和，这里规定，在第一次询问之前，xorsum=0。对所有
Type=1的数据，你需要将u,v异或上xorsum来得到真正的u,v。

Output

输出到文件mode.out中。
输出q行，每行两个数，分别表示当前询问的路径上出现最多的数的出现次数和出现次数最多的数。当有多个数出现次数同为最多时，你需要输出较小的那个。另外需要注意的一点是，对于每次询问，你应当将 xorsum 异或上你输出的两个数而不是其中某一个。

Sample Input 

0
6
2 1 2
4 1 4
2 4 3
1 4 1
4 4 1
4 1 4
1 2
2 3
3 4
2 5
5 6
2
3 4
1 4

Sample Output 

2 1
4 1

Data Constraint

对于前 10% 的数据，保证 n, q ≤ 10^4 。
对于前 20% 的数据，保证 n, q ≤ 23333 。
对于前 50% 的数据，保证 Type = 0 ，数据较为随机且具有一定梯度。
对于 100% 的数据，保证 1 ≤ n ≤ 8×10^4, 1 ≤ q ≤ 10^5, Type∈ {0, 1},1 ≤ ai,1, ai,2, ai,3 ≤ n
本题采用捆绑测试，前 10 个子任务均为 5 分，最后一个子任务为 50 分。
